# 产品需求文档 (PRD)：墨匣 MarkBox
## 1. 产品概述
- **产品名称：** 墨匣 MarkBox
- **产品定位：** 面向 AI Agent 与自动化工作流的“专属通知阅读器”。
- **核心理念：** **“发送是 AI 的脏活累活，阅读是人的高光时刻。”** 彻底抛弃传统邮箱“人-人”双向沟通的交互模型，专为接收、解析、高保真还原机器生成的复杂内容（Markdown/HTML/代码/图表）而生。
## 2. 核心痛点与解决方案
| 痛点 | 解决方案 |
| :--- | :--- |
| 传统客户端将 AI 发来的结构化报告压扁成丑陋的纯文本。 | **高保真渲染引擎**：深度剥离 MIME，优先提取 MD，支持代码高亮与复杂排版还原。 |
| AI 生成的流程图/代码在邮箱里变成一堆无法阅读的乱码。 | **图表识别与转换**：识别 Mermaid 等图表语法并转为可视化视图。 |
| 传统邮箱一半 UI 在诱导“写回复”，对单向通知是严重噪音。 | **绝对只读**：移除所有撰写入口，界面 100% 让位给“阅读内容”。 |
| AI 机器人的发件人名字经常是一串晦涩的 UUID。 | **Agent 身份强化**：强制提取并放大展示 From Name，辅以机器标识。 |
## 3. 技术栈选型
- **UI 框架：** Flutter (仅编译 Android APK)
- **状态管理：** Riverpod
- **邮件协议与解析：** `enough_mail` (处理 IMAP 连接与底层 MIME 拆解)
- **内容渲染引擎：**
  - `flutter_markdown` + `markdown` (MD 基础渲染)
  - `highlight` 或 `flutter_highlight` (代码块语法高亮)
  - `flutter_html` (HTML 兜底渲染)
- **本地存储：** `shared_preferences` (仅存账号配置，不做重度本地缓存，保持轻量)

## 4. 功能需求详情
### F1: 账户配置模块
- **F1.1 IMAP 配置页：** 表单输入 IMAP 服务器地址、端口、邮箱账号、授权码。
- **F1.2 连接测试：** 保存前进行 IMAP 登录鉴权，成功后持久化配置。
### F2: 通知流列表模块
- **F2.1 数据拉取：** 调用 `enough_mail` 拉取收件箱最新 50 封邮件（按时间倒序）。
- **F2.2 列表项展示：**
  - **第一行（重点）：** 独占一行，加粗显示**发件人显示名**。如果为空则降级显示邮箱地址。
  - **第二行：** 邮件主题。
  - **第三行：** 相对时间（如“2小时前”）。
- **F2.3 交互：** 支持下拉刷新，上拉加载更多。
### F3: 邮件解析引擎 - 极致剥离
*目标：无论 AI 工具用什么库发邮件，都要把最干净的“内容骨骼”剔出来。*
- **F3.1 MIME 遍历：** 解析 `MimeMessage` 多部分结构。
- **F3.2 正文提取优先级（严格执行）：**
  1. **最高优：** 寻找 `text/markdown`。
  2. **次高优：** 寻找 `text/html`（过滤掉邮件外壳冗余 CSS，保留核心）。
  3. **兜底：** `text/plain`。
- **F3.3 编码解码：** 强制处理 `Base64` 和 `Quoted-Printable`，杜绝乱码。
### F4: 沉浸式内容渲染模块 - 核心 UI
- **F4.1 Markdown 增强渲染（重点打磨）：**
  - 集成代码块语法高亮插件。
  - 为所有代码块右上角渲染半透明“复制”图标（点击复制并显示 Toast）。
- **F4.2 HTML 兜底渲染：** 针对 AI 常用标签（`<table>`, `<pre>`, `<code>`）做深色模式专属 CSS 适配。
- **F4.3 纯文本降级：** 使用等宽字体（如 JetBrains Mono）展示，保留换行缩进，呈现“终端美感”。
- **F4.4 异常容错：** 解析失败展示特定错误 UI，绝不闪退。

## 5. 开发里程碑 (给 Trae 的喂食顺序)
- **Phase 1：骨架与连通**
  - Prompt: "基于 Flutter/Riverpod，用 `enough_mail` 写一个 IMAP 配置页，连接服务器拉取 10 封邮件，提取 UID、发件人 Personal Name、主题，展示在 ListView。"
- **Phase 2：攻坚 MIME 解析**
  - Prompt: "实现点击列表进入详情。写解析函数遍历 MIME 结构，按优先级找 `text/markdown` -> `text/html` -> `text/plain`。确保处理 Base64/QP 解码。结果作为 Enum 返回。"
- **Phase 3：渲染引擎对接**
  - Prompt: "详情页根据 Enum 分流渲染：MD 用 `flutter_markdown`，HTML 用 `flutter_html`，纯文本用等宽字体的 `SelectableText`。加上 try-catch 兜底。"
- **Phase 4：极致打磨（代码与视觉）**
  - Prompt: "引入 highlight 库，为 Markdown 中的代码块增加语法高亮。为代码块添加一个右上角的'复制'按钮。全局强制使用深色主题。"
